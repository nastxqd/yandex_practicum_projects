# Часть 1. Аналитика Яндекс.Афиши (SQL + DataLens)

## 📌 Цель проекта
Создать комплексный аналитический дашборд для мониторинга ключевых бизнес-метрик платформы Яндекс.Афиша. Дашборд позволяет отслеживать динамику выручки, заказов и аудитории, а также анализировать структуру продаж в разрезе регионов, типов мероприятий и устройств.

### Задачи проекта
- Написание SQL-запросов для расчёта ключевых метрик
- Построение дашборда в Yandex DataLens с фильтрацией
- Визуализация недельной динамики показателей
- Анализ структуры выручки в разрезе устройств и типов мероприятий
- Выделение топ-сегментов (регионы, события, площадки)
- Формулировка рекомендаций для бизнеса

---

### Часть 1. Анализ данных с помощью SQL

**1. Расчёт общих показателей по валютам**

- Количество заказов
- Средняя выручка с заказа

- Среднее число билетов в заказе
- Средняя выручка с билета
- Доля выручки от общего объёма

**4. Недельная динамика показателей**
- Выручка
- Количество заказов
- Уникальные пользователи
- Средняя выручка с заказа

**5. Топ-7 регионов по выручке**

- Средняя выручка с билета

### Часть 2. Построение дашборда в DataLens

**Верхний блок (KPI)**
- Итоговая выручка за период
- Общее количество заказов
- Средняя выручка с заказа
- Число уникальных покупателей

**Средний блок (динамика)**
- Линейные графики по неделям:
  - выручка
  - количество заказов
  - средняя выручка с заказа

**Нижний блок (структура и топ-сегменты)**
- Столбчатые диаграммы:
  - выручка по типам устройств
  - выручка по типам мероприятий
- Таблицы:
  - топ-регионы
  - топ-события
  - топ-площадки

**Фильтры**
- по валюте заказа (по умолчанию — рубли)
- по дате (по умолчанию: 01.06.2024 – 31.10.2024)
- по типу событий
- по региону

# Часть 2: Исследовательский анализ данных в Python и проверка гипотез. 

## 📌 Цель проекта
Провести исследовательский анализ данных и проверить статистические гипотезы о поведении пользователей Яндекс.Афиши, чтобы выявить изменения в предпочтениях в осенний период и определить ключевые факторы, влияющие на активность.

### Задачи проекта
- Загрузка и предобработка данных (пропуски, дубликаты, выбросы, конвертация валют)
- Исследовательский анализ данных (EDA)
- Анализ временных трендов
- Сравнение поведения пользователей разных типов устройств
- Проверка статистических гипотез (U-тест Манна-Уитни)
- Формулировка выводов и рекомендаций

---

## ⚙️ Этапы проекта

### Шаг 1. Загрузка и знакомство с данными
- Загрузка трёх датасетов
- Проверка размеров, структуры, типов данных
- Первичный осмотр и выявление потенциальных проблем

### Шаг 2. Предобработка данных
- **Пропуски:** только `days_since_prev` (не аномалия)
- **Категориальные переменные:** проверка уникальных значений
- **Количественные переменные:** анализ выбросов по процентилям, фильтрация по 99-му процентилю
- **Дубликаты:**
  - по времени: 85 неявных дубликатов → удалены
  - по дате: много дубликатов (пользователи покупают несколько билетов в день) → оставлены
- **Преобразование типов:** даты → datetime
- **Конвертация валют:** создан `revenue_rub` (курс тенге / 100)
- **Новые признаки:**
  - `one_ticket_revenue_rub` — выручка с билета
  - `month` — месяц заказа
  - `season` — сезон (лето/осень/зима/весна)

### Шаг 3. Исследовательский анализ данных (EDA)

#### 3.1. Сезонные изменения
- Количество заказов растёт с июня по ноябрь
- **Мобильные устройства:** ~80% заказов в любом сезоне
- **Возрастные категории:** летом лидирует 16+ (28,4%), осенью догоняет 0+ (23,6%)
- **Типы мероприятий:** концерты (37–42%), театр (20–25%)
- **Цены:** осенью снизились у театра, ёлок, концертов; выросли у стендапа и выставок

#### 3.2. Осенняя активность пользователей
- **Динамика по дням:** устойчивый рост с сентября по октябрь
- **Пики:** 03.09.2024 и 01.10.2024 (аномально низкие цены <100 руб.)
- **Недельная цикличность:**
  - заказов больше в будни (люди планируют выходные заранее)
  - DAU почти одинаков в будни и выходные
- **Аномалии:** вторник — возможная акция

#### 3.3. Популярные события и партнёры
- **Регионы:** Каменевский (24,9% мероприятий), Североярская область (16,7%)
- **Партнёры по выручке:** «Билеты без проблем» (16,7%), «Мой билет» (14,1%), «Облачко» (12,5%)
- **Концентрация:** 43,3% выручки от трёх партнёров

### Шаг 4. Статистический анализ данных

#### Гипотеза 1: Среднее количество заказов на пользователя
- **H₀:** среднее для mobile ≤ среднее для desktop
- **H₁:** среднее для mobile > среднее для desktop
- **Тест:** U-тест Манна-Уитни (распределения ненормальные)
- **Результат:** p-value < 0.05 → **отвергаем H₀**
- **Вывод:** мобильные пользователи совершают статистически значимо больше заказов

#### Гипотеза 2: Среднее время между заказами
- **H₀:** время между заказами для mobile ≥ время для desktop
- **H₁:** время между заказами для mobile < время для desktop
- **Тест:** U-тест Манна-Уитни
- **Результат:** p-value ≥ 0.05 → **принимаем H₀**
- **Вывод:** статистически значимых различий во времени между заказами не обнаружено

## 📊 Результаты

### Ключевые выводы

| Категория | Вывод |
|-----------|-------|
| **Популярность мероприятий** | Концерты — 37–42%, театр — 20–25%. Осенью театр растёт, концерты снижаются |
| **Цены** | Осеннее снижение у большинства категорий, кроме стендапа (+6%) |
| **Мобильное доминирование** | 80% заказов с мобильных устройств |
| **Активность мобильных** | Больше заказов на пользователя (статистически значимо) |
| **Регионы** | Каменевский регион — 25% мероприятий |
| **Партнёры** | 43,3% выручки от трёх партнёров |

### Проверка гипотез

| Гипотеза | Результат | p-value |
|----------|-----------|---------|
| Мобильные пользователи делают больше заказов | ✅ Подтверждена | < 0.05 |
| Мобильные пользователи покупают чаще (меньше интервал) | ❌ Не подтверждена | ≥ 0.05 |
