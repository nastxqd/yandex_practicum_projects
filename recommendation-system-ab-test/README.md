# Разработка A/B-тестирования и анализ результатов

## 📌 Цель проекта
Рассчитать параметры A/B-теста, оценить корректность его проведения и проанализировать результаты эксперимента по внедрению нового алгоритма рекомендаций в развлекательном приложении.

### Задачи проекта:
1. Провести исследовательский анализ исторических данных приложения
2. Сформулировать нулевую и альтернативную гипотезы
3. Рассчитать необходимый размер выборки и длительность теста
4. Проверить корректность проведения A/B-теста (распределение пользователей, независимость групп, баланс по категориальным признакам)
5. Проанализировать результаты эксперимента
6. Сделать вывод о целесообразности внедрения нового алгоритма

---

## ⚙️ Этапы проекта

### Шаг 1. Работа с историческими данными (EDA)
- Анализ числа регистраций и динамики пользователей
- Анализ числа просмотренных страниц
- Расчёт доли успешных сессий (≥4 страниц)

### Шаг 2. Подготовка к тесту
- Формулировка нулевой и альтернативной гипотез
- Расчёт необходимого размера выборки
- Определение длительности A/B-теста

### Шаг 3. Мониторинг A/B-теста
- Проверка распределения пользователей по группам
- Проверка независимости выборок (отсутствие пересечений)
- Проверка равномерности распределения по устройствам
- Проверка равномерности распределения по регионам

### Шаг 4. Проверка результатов A/B-теста
- Подсчёт основной метрики (доля успешных сессий)
- Статистическая проверка корректности теста
- Сравнение доли успешных сессий между группами
- Оценка статистической значимости изменений

## 📊 Результаты

### Исторический анализ данных

#### Динамика пользователей
- Период наблюдения: 11.08.2025 — 23.09.2025
- Доля зарегистрированных пользователей стабильна: 8-11%

#### Распределение сессий по количеству страниц

| Количество страниц | Количество сессий |
|-------------------|------------------|
| 1 | ~29,000 |
| 2 | ~105,000 |
| 3 | ~160,000 (пик) |
| 4 | ~105,000 |
| 5 | ~50,000 |
| 6-7 | < 10,000 |

#### Доля успешных сессий (≥4 страниц)
- Среднее значение: 30%
- Диапазон колебаний: 28.9% — 32%
- Отсутствие резких скачков — стабильное пользовательское поведение

### Параметры A/B-теста

| Параметр | Значение |
|----------|---------|
| Целевая метрика | Доля успешных сессий (≥4 страниц) |
| Базовый уровень | 30% |
| Минимальный детектируемый эффект (MDE) | 3% (относительно базового уровня) |
| Уровень значимости (α) | 0.05 |
| Мощность теста (1-β) | 0.8 |
| Необходимый размер выборки на группу | ~12,000 пользователей |
| Средний дневной трафик | ~3,100 пользователей |
| Расчётная длительность теста | 8 дней |

### Проверка корректности теста

| Проверка | Результат |
|----------|-----------|
| Распределение пользователей | Группа A: 1,477 / Группа B: 1,466 |
| Пересечение групп | Нет пересечений (выборки независимы) |
| Равномерность по устройствам | Максимальная разница 1.7% (iPhone) |
| Равномерность по регионам | Максимальная разница 0.4% (CIS) |
| Количество дневных сессий | p-value = 0.9382 > 0.05 (различий нет) |

### Результаты A/B-теста

| Показатель | Группа A (контрольная) | Группа B (тестовая) | Изменение |
|-----------|------------------------|---------------------|-----------|
| Доля успешных сессий | 31.17% | 32.27% | +1.10 п.п. |

#### Статистическая значимость
- **p-value**: 0.000157
- **Вывод**: p-value < 0.05 → статистически значимые изменения обнаружены
- **Заключение**: Отвергаем нулевую гипотезу H₀

## 📈 Ключевые выводы

### Формулировка гипотез
- **H₀ (нулевая гипотеза)**: новый алгоритм не повлияет на долю успешных сессий (p(старая) = p(новая))
- **H₁ (альтернативная гипотеза)**: новый алгоритм увеличит долю успешных сессий (p(старая) < p(новая))

### Характеристики эксперимента
- **Период теста**: 20 дней (2025-10-14 — 2025-11-02)
- **Участники**: 2,943 пользователя (A: 1,477 / B: 1,466)
- **Целевая метрика**: доля успешных сессий (≥4 просмотренных страниц)
- **Уровень значимости**: α = 0.05

### Основные результаты
✅ **Новый алгоритм рекомендаций статистически значимо увеличивает долю успешных сессий** на 1.10 процентных пункта

✅ **Изменение не является случайным**: p-value = 0.000157 < 0.05

✅ **Тест проведён корректно**: группы сбалансированы, пересечений нет, распределение по устройствам и регионам равномерно
