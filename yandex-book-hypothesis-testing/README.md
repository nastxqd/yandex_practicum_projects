# Проверка гипотез в Python и анализ результатов A/B-теста

## 📌 Цель проекта
Провести статистический анализ активности пользователей сервиса Яндекс Книги в Москве и Санкт-Петербурге, а также оценить результаты A/B-тестирования нового интерфейса интернет-магазина BitMotion Kit.

### Задачи проекта:

**Часть 1. Проверка гипотезы в Python**
1. Изучение и подготовка данных о пользователях Яндекс Книг
2. Проверка качества данных (дубликаты, выбросы, распределения)
3. Формулировка статистических гипотез
4. Выбор и проведение статистического теста
5. Интерпретация результатов и подготовка аналитической записки

**Часть 2. Анализ результатов A/B-тестирования**
1. Загрузка и оценка целостности данных
2. Проверка корректности проведения теста
3. Анализ пользовательской активности
4. Оценка достаточности выборки
5. Статистическая проверка изменения конверсии
6. Формулировка выводов и рекомендаций

## ⚙️ Этапы проекта

### Часть 1. Проверка гипотезы

#### 1. Загрузка и первичный осмотр данных
- Проверка дубликатов в идентификаторах пользователей
- Выявление пересекающихся пользователей между городами

#### 2. Очистка данных
- Удаление пользователей, встречающихся в обоих городах
- Анализ выбросов методом IQR

#### 3. Исследовательский анализ
- Сравнение размеров групп
- Анализ распределений (QQ-plot)
- Проверка нормальности распределений

#### 4. Статистическая проверка гипотезы
- Тест Левена для проверки равенства дисперсий
- Выбор типа t-теста
- Проведение одностороннего t-теста
- Интерпретация p-value

### Часть 2. Анализ A/B-теста

#### 1. Оценка корректности теста
- Проверка равномерности распределения по группам
- Выявление пересечений с конкурирующим тестом
- Очистка данных от пересекающихся пользователей

#### 2. Подготовка данных для анализа
- Определение горизонта анализа (первые 7 дней после регистрации)
- Фильтрация событий в рамках теста
- Расчёт LTV (времени жизни события)

#### 3. Оценка достаточности выборки
- Расчёт MDE (минимального детектируемого эффекта)
- Проверка соответствия заданным параметрам (мощность 80%, достоверность 95%)

#### 4. Анализ конверсии
- Расчёт количества посетителей и покупателей по группам
- Сравнение конверсии между группами

#### 5. Статистическая проверка
- Z-тест для пропорций
- Формулировка нулевой и альтернативной гипотез
- Интерпретация результатов

## 📊 Результаты

### Часть 1. Яндекс Книги

#### Очистка данных
- Исходное количество пользователей: 13,670
- Удалено пересекающихся пользователей (Москва и СПб): 244
- Итоговые группы:
  - Москва: 9,142 пользователя
  - Санкт-Петербург: 4,284 пользователя

#### Выбросы
| Город | Доля выбросов | Границы IQR |
|-------|---------------|-------------|
| Москва | 14.5% | [0.50, 20.50] |
| Санкт-Петербург | 1.3% | [0.50, 16.50] |

#### Статистики активности

| Показатель | Москва | Санкт-Петербург |
|------------|--------|-----------------|
| Среднее (часов) | 7.89 | 7.47 |
| Медиана (часов) | 1.00 | 1.00 |
| Общая активность | 72,110 | 31,981 |

#### Результат проверки гипотезы

| Параметр | Значение |
|----------|---------|
| Тип теста | Стандартный t-тест для независимых выборок |
| Уровень значимости (α) | 0.05 |
| p-value | 0.3264 |
| Результат | Невозможно отвергнуть H₀ |

**Вывод:** Статистически значимых доказательств того, что пользователи из Санкт-Петербурга проводят больше времени в приложении, чем пользователи из Москвы, не обнаружено.

#### Возможные причины
1. Схожие паттерны поведения в крупных мегаполисах
2. Распределение активности сильно скошено (большинство пользователей малоактивны)
